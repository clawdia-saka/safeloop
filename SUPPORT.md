# Support

Use GitHub Issues for reproducible bugs, documentation gaps, and scoped feature
requests:

https://github.com/clawdia-saka/safeloop/issues

For security-sensitive reports, do not open a public issue with details. Follow
`SECURITY.md` instead.

SafeLoop is currently local-first and early-stage. Public support should keep
the recovery boundary explicit: exact rollback is limited to covered local file
changes, while actions outside the local repo require compensation or manual
review.
