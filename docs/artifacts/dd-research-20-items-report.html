20項目チェックリスト

  1. 対象概要: 何を評価するか
  2. 問題設定: 解決する課題の明確さ
  3. 市場 / ユーザー: 規模・痛み・利用頻度
  4. プロダクト: 機能、成熟度、差別化
  5. 技術アーキテクチャ: 実装妥当性と依存
  6. データ / ソース: 入手元、鮮度、検証性
  7. チーム / 実行力: 経歴、速度、継続性
  8. トラクション: 利用・売上・コミュニティ
  9. 競合: 代替手段と優位性
  10. ビジネスモデル: 収益化と単価
  11. 財務 / 資金: ランウェイ、調達、コスト
  12. 法務 / 規制: 管轄、ライセンス、制約
  13. セキュリティ: 脅威、監査、インシデント
  14. 運用リスク: SLA、障害対応、権限管理
  15. ガバナンス: 意思決定と透明性
  16. パートナー / エコシステム: 依存と強み
  17. ロードマップ: 実現可能性とマイルストーン
  18. レッドフラグ: 矛盾、未証明、過大主張
  19. アップサイド: 成功シナリオと触媒
  20. 結論 / Next steps: 判断、条件、追加検証
