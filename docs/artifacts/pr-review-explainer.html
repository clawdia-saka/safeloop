3. Independent review checklist

  • Re-read the diff without relying on the author summary.
  • Verify all external-facing claims against tests, docs, or generated artifacts.
  • Confirm HTML is static and does not become a second canonical data store.
  • Check accessibility basics: title, language, headings, table headers, readable contrast.
  • Record verdict and any residual risks in the PR body.

4. Merge notes

Merge only after tests pass, the PR body includes scope/risk notes, and out-of-scope items are explicitly deferred.
