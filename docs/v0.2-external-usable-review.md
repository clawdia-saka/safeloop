# SafeLoop v0.2 external-usable review checklist (#109-#115)

This checklist is for reviewing PRs that touch the External-usable roadmap path. It is intentionally offline-first: SafeLoop may record, summarize, and guide external side-effect review, but this harness must not introduce live third-party integrations.

## Current gate: #109 operator-packet CLI + external registry

- [ ] `safeloop external record` appends deterministic `external-effects.jsonl` records.
- [ ] Every external side-effect record has `exact_rollback: false`.
- [ ] Evidence is a stable path/URL/reference plus a minimal quote/field, not a raw payload dump.
- [ ] Operator packet output separates:
  - covered local file rollback (`Exact rollback: true`), from
  - external compensation/manual-review rows (`Exact rollback: false`).
- [ ] Recommended next action becomes `compensation_review_required` when external effects exist.
- [ ] No raw secrets, tokens, bearer headers, full webhook payloads, email bodies, or personal data are stored in review artifacts.

## Roadmap artifact gates (#111-#115)

### #111 compensation plan

- [ ] Compensation plan is an explicit artifact (for example `compensation-plan.json`) and is linked from operator-facing output.
- [ ] Plan entries reference `external-effects.jsonl` effect IDs/evidence, not live service state as the source of truth.
- [ ] Plans use capability language (`none`, `manual`, `best_effort`, `verified`) and do not claim exact rollback.
- [ ] Manual handoff is acceptable and must be visibly distinct from automated compensation.

### #112 compensation receipt/result

- [ ] Compensation result/receipt is a separate artifact from the plan.
- [ ] Receipts record outcome, operator/action identity where appropriate, timestamp, and evidence references.
- [ ] Receipts do not include sensitive raw external payloads.
- [ ] Partial or failed compensation remains visible as review-required; it is not hidden by local rollback success.

### #113 external adapter interface

- [ ] Adapter contracts are fake/offline-testable by default.
- [ ] Adapter interfaces expose idempotency/evidence boundaries and compensation capability without dispatching live external effects in tests.
- [ ] Unit/E2E tests use fakes, fixtures, or local files only.
- [ ] Exact external rollback is not added as an adapter capability.

### #114 fake webhook demo

- [ ] Demo uses an offline/local fake webhook or recorded fixture only.
- [ ] Demo output proves what would be sent/compensated without contacting a live endpoint.
- [ ] Evidence is redacted/minimal and deterministic.
- [ ] Demo docs make clear that fake webhook behavior is not a production integration.

### #115 operator packet integration

- [ ] Operator packet links external effect, compensation plan, and compensation result artifacts when present.
- [ ] Packet keeps rollback, compensation, and manual-review sections visibly separate.
- [ ] Packet states that exact rollback applies only to covered local file changes.
- [ ] Packet provides operator next steps for unresolved external effects.
- [ ] Packet remains deterministic under repeated generation from the same run directory.

## Required test harness usage

Run the external-usable harness before review approval:

```bash
pytest -q tests/test_external_effects.py tests/test_operator_packet_cli.py tests/test_operator_packet_v2.py tests/test_external_usable_e2e.py
git diff --check
```

`tests/test_external_usable_e2e.py` now contains concrete assertions for the linked external-usable artifact path: external-effect registry, compensation plan/result receipt, operator packet rows/links, operator packet manifest verification, and `exact_rollback: false` for external actions. Adapter-interface and fake-demo contract details remain covered by their narrower docs/tests rather than by this single E2E test.
