# SafeLoop v0.2.0 RC definition of done

This document is the release-candidate definition of done for SafeLoop v0.2.0. It is a documentation and readiness gate: it records what must be true before the RC is described as done, and what remains explicitly excluded from the release.

## Scope statement

SafeLoop v0.2.0 RC is done only for the local recoverability story. Exact rollback is limited to covered local file changes captured in SafeLoop artifacts. External side effects are observable and reviewable through registry, compensation, and operator-packet evidence, but they are not exactly rolled back by SafeLoop.

## Required RC gates

The RC is not done unless all of these gates are satisfied and linked from the readiness matrix:

1. **Local exact rollback gate**
   - `safeloop rollback plan` and `safeloop rollback apply` work for covered local file changes.
   - Selective file and hunk rollback behavior is documented and tested.
   - Documentation states that exact rollback applies only to local covered artifacts.

2. **External side-effect registry gate**
   - External effects are represented in a registry or ledger with enough evidence for operator review.
   - The registry distinguishes local rollback from remote/external action handling.
   - The docs state that external exact rollback is out of scope.

3. **Compensation plan/result gate**
   - Compensation documentation describes operator-reviewed plans and recorded results.
   - Compensation output is evidence for manual remediation, not a promise of automatic external remediation.
   - Plan/result examples remain aligned with the recovery story.

4. **Compensation adapter contract gate**
   - Adapter/fake examples demonstrate the expected request/result shape without invoking real external services.
   - Adapter contracts include operator boundary language and do not imply production third-party adapter support.

5. **Fake external demo gate**
   - The demo uses fake/local external-effect artifacts so it can run without real GitHub, messaging, email, webhook, or hosted-service side effects.
   - Demo docs show how an operator reviews external side-effect evidence and compensation records.

6. **Operator packet integration gate**
   - The operator packet includes rollback, external side-effect, and compensation evidence needed for handoff.
   - The packet keeps local exact rollback separate from compensation/manual review for external actions.

7. **Readiness-matrix/audit gate**
   - The readiness matrix names the required gates and their evidence.
   - The completion gap audit and this DoD document repeat the release exclusions so they are visible during review.

## Required exclusions

The v0.2.0 RC must not claim or imply any of the following:

- **No real external adapters**: v0.2.0 RC includes adapter contracts and fake/demo artifacts only; it does not ship production adapters for real third-party services.
- **No external exact rollback**: actions outside the local repo are not exactly rolled back by SafeLoop.
- **No hosted control plane**: v0.2.0 RC does not ship a hosted multi-user control plane.
- **No automatic external remediation**: compensation plans/results are operator evidence; SafeLoop does not automatically remediate external systems in this RC.

## RC done checklist

- [ ] Local exact rollback docs and tests are present.
- [ ] External side-effect registry docs/tests are present.
- [ ] Compensation plan/result docs/tests are present.
- [ ] Compensation adapter contract and fake demo docs/tests are present.
- [ ] Operator packet integration docs/tests are present.
- [ ] Readiness matrix lists the gates above.
- [ ] Docs repeat the four required exclusions: no real external adapters, no external exact rollback, no hosted control plane, and no automatic external remediation.

## Review rule

If a future PR adds a real external adapter, hosted control plane, automatic external remediation, or exact rollback for a remote service, it must update this DoD and the readiness matrix in the same PR. Until then, those capabilities remain outside the v0.2.0 RC definition of done.
