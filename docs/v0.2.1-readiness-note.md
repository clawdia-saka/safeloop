# SafeLoop v0.2.1 candidate readiness note

This note documents the current v0.2.1 candidate release posture. It is a readiness and loop-suppression note only. It does not create a tag, GitHub Release, or PyPI publish, and it does not change runtime behavior, rollback behavior, or compensation semantics.

## 1. Status summary

- Ready for public-readiness messaging: YES
- Ready for tag / GitHub Release / PyPI: HOLD
- Reason for HOLD: TT explicit approval required before release/publish

Current verification baseline:

- `python scripts/public_readiness.py --check` reports `public-readiness: ok`.
- Focused readiness/operator packet tests pass.
- `pytest -q` passes with the expected xfail.
- exact_rollback remains false for external actions.

## 3. Current packet behavior

After verified manual compensation for a recorded external effect, the operator packet is expected to show coherent current-state rows:

- primary external-effect row after verified manual compensation: `compensation_completed`
- sibling `manual_review_item` is no longer `queued`
- sibling `compensation_item` is no longer `compensation_review_required`
- recommended next action: `compensation_complete_verify_receipt`
- no exact_rollback=true promotion for external actions

## 4. Boundary

- exact rollback only for covered local file changes
- external actions remain exact_rollback=false
- External side effects are compensation/manual-review only
- no hosted control plane
- no remote transparency log
- no PyPI/GitHub Release/tag without explicit TT approval

This candidate does not claim exact rollback for actions outside the local repo. External actions remain documented as compensation/manual-review flows, backed by local artifacts and receipt evidence where completion is shown.

## 6. Packet hash verification gap

`packet_hash_verify_design_gap` is addressed by the local `operator-packet-manifest.v1` manifest and `safeloop operator-packet-verify` command. This adds local packet/source hash verification for `operator-packet-v2.md` and its source artifacts. It is tamper-evident local-only evidence, not a hosted control plane, not a remote transparency log, and not a tamper-proof guarantee.

External actions remain `exact_rollback=false`; the manifest does not change rollback or compensation semantics.
